## 架构说明

### 1. 客户端 & 渠道 (Clients & Channels)
这是用户与 OpenClaw 交互的界面。
*   **原生应用**: iOS, Android, macOS 应用通过 API 与网关通信。
*   **IM 平台**: Slack, Discord, Telegram 等通过各自的 API 或 Webhook 与 OpenClaw 集成。

### 2. 网关接入层 (Gateway Layer)
负责处理所有入站和出站流量。
*   **Channel Docks**: `src/channels/dock.ts` 定义了统一的接口，各个渠道（如 `src/slack/`, `src/discord/`）实现了具体的适配逻辑，将不同平台的格式转换为 OpenClaw 内部消息格式。
*   **API Server**: `src/gateway/server.ts` 提供 HTTP 和 WebSocket 接口，供客户端应用使用。
*   **Auth**: `src/gateway/auth.ts` 处理认证和权限校验。

### 3. 核心服务层 (Core Services)
系统的骨架，负责消息流转和状态管理。
*   **Router**: 决定消息应该发送给哪个 Agent 或处理模块。
*   **Session Manager**: `src/config/sessions.ts` 管理用户会话状态。
*   **Config Manager**: `src/config/config.ts` 加载和验证系统配置。

### 4. 代理智能层 (Agent Layer)
OpenClaw 的大脑。
*   **Agent Engine**: 协调上下文、模型和工具调用。
*   **Context Manager**: `src/agents/context.ts` 管理 LLM 的上下文窗口，确保不超过 Token 限制。
*   **LLM Interface**: 对接不同的模型提供商（如 Anthropic, OpenAI）。

### 5. 能力扩展层 (Capability Layer)
赋予 Agent 执行实际任务的能力。
*   **Skills**: `skills/` 目录下包含具体的功能实现（如查询天气、操作 GitHub）。
*   **Plugin Runtime**: `src/plugins/runtime.ts` 支持动态加载插件。
*   **Sandbox**: `src/agents/sandbox.ts` 提供安全的执行环境，防止恶意代码破坏系统。

### 6. 基础设施层 (Infrastructure)
底层支持服务。
*   **SQLite**: `src/memory/sqlite.ts` 提供持久化存储。
*   **Logging**: `src/logging/` 提供结构化日志记录。
