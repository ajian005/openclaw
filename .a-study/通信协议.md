# 通信协议 (Communication Protocol)

OpenClaw 使用多种协议与客户端、浏览器扩展和内部组件进行通信。

## 端口概览

| 端口 | 用途 | 协议 | 说明 |
| :--- | :--- | :--- | :--- |
| **18789** | **Gateway 主服务** | HTTP / WebSocket | 所有客户端（CLI, Web, App）的主要连接点。 |
| **18790** | **Browser Control** | HTTP | 用于 Agent 控制浏览器的指令接口。 |
| **18792** | **Chrome Extension** | WebSocket | 浏览器扩展专用的连接端口。 |

## Gateway 主协议 (Port 18789)

### WebSocket API
主要用于实时双向通信。
*   **路径**: `/` (根路径)
*   **消息格式**: JSON-RPC 风格
    *   **请求**: `{ "id": "req-1", "method": "chat.send", "params": { ... } }`
    *   **响应**: `{ "id": "req-1", "result": { ... } }`
    *   **事件**: `{ "event": "agent.chunk", "data": { ... } }`

### HTTP API
提供标准的 RESTful 接口。
*   **OpenAI 兼容接口**:
    *   `POST /v1/chat/completions`: 支持标准 OpenAI SDK 调用 OpenClaw 的 Agent。
*   **工具调用**:
    *   `POST /tools/invoke`: 直接调用某个 Skill 工具。
*   **健康检查**:
    *   `GET /health`: 返回系统健康状态。

## 浏览器控制协议 (Port 18790 & 18792)

### Agent -> Server (HTTP 18790)
Gateway 通过 HTTP 请求向浏览器控制服务发送指令。
*   `POST /browse/open`: 打开 URL。
*   `POST /browse/act`: 执行点击、输入等操作。

### Server -> Extension (WebSocket 18792)
控制服务通过 WebSocket 将指令推送到 Chrome 扩展。
*   **Protocol**: 自定义 JSON 消息。
*   **指令**: `navigate`, `click`, `type`, `scroll`, `screenshot`.

## 认证机制

OpenClaw 支持基于 Token 的认证。
*   **HTTP Header**: `Authorization: Bearer <token>`
*   **WebSocket Query**: `?token=<token>`
*   **Token 来源**: 启动时自动生成或在 `openclaw.toml` 中配置 `gateway.auth.token`。
