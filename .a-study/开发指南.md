# 开发指南

欢迎参与 OpenClaw 开发！本指南将帮助你快速上手项目构建、测试和扩展。

## 1. 环境准备

OpenClaw 是一个 Monorepo 项目，主要使用 TypeScript。

*   **Node.js**: 推荐使用最新的 LTS 版本 (v20+)。
*   **包管理器**: 必须使用 **pnpm**。
    *   安装: `npm install -g pnpm`
*   **其他工具**:
    *   `oxlint` / `oxfmt`: 用于代码检查和格式化。
    *   `docker`: 如果你需要运行沙箱环境或特定服务。

## 2. 快速开始

### 安装依赖
```bash
pnpm install
```

### 构建项目
```bash
pnpm build
```
这将编译 TypeScript 代码，处理 assets，并生成构建产物。

### 运行开发模式
启动 Gateway 和 CLI 进行调试：
```bash
pnpm dev
# 或者指定子命令
pnpm dev gateway
```

### 运行测试
```bash
# 运行所有测试
pnpm test

# 运行静态检查
pnpm check
```

## 3. 核心开发任务

### 添加一个新的 Channel
1.  在 `src/` 下创建一个新目录，例如 `src/my-channel/`.
2.  实现 `ChannelDock` 接口 (`src/channels/dock.ts`).
3.  实现接收消息逻辑 (Webhook 或 Polling).
4.  实现发送消息逻辑 (`send.ts`).
5.  在 `src/gateway/server-channels.ts` 或 `src/channels/registry.ts` 中注册你的 Channel。

### 添加一个新的 Skill
1.  在 `skills/` 目录下创建新目录，例如 `skills/my-skill/`.
2.  创建 `SKILL.md` (或类似定义文件)，描述技能的元数据和工具定义。
3.  编写技能的实现代码（通常是 TypeScript 或 Python 脚本）。
4.  在 `src/agents/skills.ts` 或相关配置中确保技能被加载。

### 修改 Agent 行为
*   **Prompt**: 修改 `src/agents/identity.ts` 或配置文件中的 System Prompt。
*   **Context**: 调整 `src/agents/context.ts` 中的上下文逻辑。

## 4. 项目结构规范

*   **`src/cli/`**: CLI 命令定义。所有新的命令行功能应在这里添加。
*   **`src/gateway/`**: 服务器端逻辑。不要在这里写具体的业务逻辑，保持 Gateway 的通用性。
*   **`src/channels/`**: 渠道适配器。保持与 Gateway 的解耦。

## 5. 提交代码

*   请遵循 `CONTRIBUTING.md` 中的规范。
*   确保 `pnpm check` 和 `pnpm test` 通过。
*   PR 标题清晰描述变更内容。

## 6. 调试技巧

*   **日志**: OpenClaw 使用结构化日志。在开发模式下，日志会输出到控制台。
*   **调试器**: 可以使用 VS Code 的 "JavaScript Debug Terminal" 运行 `pnpm dev` 进行断点调试。
*   **Web 控制台**: 启动 Gateway 后，访问 `http://localhost:18789` (默认端口) 查看系统状态。
