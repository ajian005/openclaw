# OpenClaw 项目代码结构分析

OpenClaw 是一个多渠道 AI 网关，支持扩展的消息集成。项目采用 TypeScript (Node.js) 编写，核心逻辑位于 `src/` 目录下，同时包含移动端和桌面端应用 (`apps/`) 以及丰富的文档 (`docs/`)。

## 目录结构概览

### 1. 根目录
*   **配置文件**:
    *   `package.json`: 项目依赖、脚本定义。
    *   `pnpm-workspace.yaml`: Monorepo 工作区配置。
    *   `Dockerfile`, `docker-compose.yml`: 容器化部署配置。
    *   `fly.toml`: Fly.io 部署配置。
*   **`openclaw.mjs`**: CLI 的二进制入口文件。
*   **`scripts/`**: 包含构建、测试、部署等实用脚本 (e.g., `run-node.mjs`, `build_icon.sh`)。
*   **`docs/`**: 详细的项目文档，包含各功能模块的说明。

### 2. 核心源码 (`src/`)
这是项目的核心后端逻辑所在。

#### 2.1 命令行接口 (`src/cli/`)
负责处理命令行交互和启动各类服务。
*   **`program.ts`**: CLI 程序构建入口。
*   **`*-cli.ts`**: 各个子命令的实现文件，例如：
    *   `acp-cli.ts`: 代理控制协议 (Agent Control Protocol) 相关命令。
    *   `daemon-cli.ts`: 守护进程管理。
    *   `gateway-cli.ts`: 网关服务启动。
    *   `config-cli.ts`: 配置管理。
    *   `skills-cli.ts`: 技能管理。

#### 2.2 代理与智能 (`src/agents/`)
定义 AI 代理的行为和上下文。
*   **`context.ts`**: 管理模型上下文窗口，加载模型配置。
*   **`identity.ts`**: 代理身份标识。
*   **`skills.ts`**: 技能加载与执行逻辑。
*   **`sandbox.ts`**: 代理执行的安全沙箱环境。

#### 2.3 渠道集成 (`src/channels/` & 根级特定目录)
处理不同通信平台的对接。
*   **`dock.ts` (`src/channels/`)**: 渠道的统一接入点和注册表，定义了 `ChannelDock` 接口。
*   **具体渠道实现**:
    *   `src/discord/`: Discord API 集成。
    *   `src/slack/`: Slack API 集成。
    *   `src/telegram/`: Telegram Bot 集成。
    *   `src/line/`: LINE Bot 集成。
    *   `src/signal/`: Signal 消息集成。
    *   `src/imessage/`: iMessage 集成。
    *   `src/browser/`: 浏览器自动化/控制集成。

#### 2.4 配置管理 (`src/config/`)
*   **`config.ts`**: 核心配置加载逻辑。
*   **`schema.ts`**: 配置文件的结构定义。
*   **`env-vars.ts`**: 环境变量处理。
*   **`sessions.ts`**: 会话存储管理。

#### 2.5 基础设施 (`src/infra/`)
提供底层通用服务。
*   **`net/`**: 网络相关工具。
*   **`logging/`**: 日志系统 (`src/logging/`).
*   **`fs-safe.ts`, `json-file.ts`**: 文件系统操作。
*   **`exec-host.ts`**: 进程执行封装。

#### 2.6 网关服务 (`src/gateway/`)
*   **`server.ts`**: HTTP/WebSocket 服务器实现。
*   **`auth.ts`**: 认证逻辑。
*   **`routes`**: API 路由定义。

#### 2.7 记忆与存储 (`src/memory/`)
*   **`manager.ts`**: 记忆管理器。
*   **`sqlite.ts`**: SQLite 数据库适配器，用于持久化存储。

#### 2.8 插件系统 (`src/plugins/`, `src/plugin-sdk/`)
*   **`loader.ts`**: 插件加载器。
*   **`runtime.ts`**: 插件运行时环境。

### 3. 技能 (`skills/`)
位于根目录的 `skills/` 文件夹包含了具体的技能实现，每个子目录代表一个技能：
*   `github/`: GitHub 集成技能。
*   `trello/`: Trello 管理技能。
*   `weather/`: 天气查询技能。
*   `notion/`: Notion 集成技能。
*   等等。

### 4. 客户端应用 (`apps/`)
*   `android/`: Android 客户端项目。
*   `ios/`: iOS 客户端项目。
*   `macos/`: macOS 客户端项目。

## 关键流程推断

1.  **启动流程**: 用户运行 `openclaw` 命令 -> `openclaw.mjs` -> `src/index.ts` -> `src/cli/program.ts` -> 根据参数分发到对应的 `*-cli.ts` (如 `gateway-cli.ts`)。
2.  **消息处理**: 外部消息 -> 特定渠道适配器 (e.g., `src/slack/`) -> `src/gateway/` -> 路由分发 -> `src/agents/` 处理 -> 调用 `skills/` 或 LLM -> 返回响应。
3.  **扩展机制**: 通过 `src/plugins/` 加载插件，或通过配置添加新的 `skills`。
