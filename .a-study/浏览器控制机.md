# 浏览器控制 (Browser Control)

OpenClaw 具备控制用户浏览器执行操作的能力，这通过一个专用的 Chrome 扩展和本地 WebSocket 服务实现。

## 核心架构

```mermaid
graph TD
    Agent[Agent (LLM)] -->|指令: open_url/click| Gateway
    Gateway -->|HTTP/WS| BrowserServer[Browser Control Server (:18790)]
    BrowserServer -->|WebSocket| Extension[Chrome Extension (:18792)]
    Extension -->|Chrome API| Tab[Browser Tab]
```

## 组件详解

### 1. Browser Control Server (`src/browser/server.ts`)
这是一个独立的 Express 服务器，通常运行在 `127.0.0.1:18790`。
*   **职责**: 作为 Agent 和 Chrome 扩展之间的桥梁。
*   **接口**: 提供 REST API 供 Gateway 调用（例如 `POST /browse/open`）。
*   **状态管理**: 维护当前连接的浏览器实例和标签页状态。

### 2. Chrome Extension (`assets/chrome-extension/`)
运行在用户浏览器中的插件。
*   **连接**: 启动时主动连接到 Browser Control Server 的 WebSocket。
*   **执行**: 接收来自 Server 的 JSON 指令，调用 `chrome.tabs`, `chrome.scripting` 等 API 操作页面。
*   **反馈**: 将页面加载状态、截图、DOM 内容回传给 Server。

## 工作流程

1.  **指令生成**: Agent 决定需要访问网页，生成工具调用指令（如 `browser.goto("https://google.com")`）。
2.  **指令转发**: Gateway 将指令转发给 Browser Control Server。
3.  **执行操作**: Server 通过 WebSocket 通知 Chrome 扩展。
4.  **操作浏览器**: 扩展在浏览器中创建新标签页或在当前标签页执行 JS。
5.  **获取结果**: 扩展抓取页面内容（文本/截图），回传给 Server。
6.  **返回上下文**: Server 将结果返回给 Gateway，最终进入 Agent 的上下文窗口。

## 安装与使用

1.  **加载扩展**:
    *   打开 Chrome 的 `chrome://extensions/`。
    *   开启 "Developer mode"。
    *   点击 "Load unpacked"，选择 `openclaw/assets/chrome-extension` 目录。
2.  **连接状态**:
    *   点击浏览器工具栏上的 OpenClaw 图标。
    *   确保显示 "Connected" 或 "ON"。
3.  **配置**:
    *   在 `openclaw.toml` 中启用浏览器控制：
        ```toml
        [browser]
        enabled = true
        headless = false # 通常设为 false 以便观察
        ```

## 安全性
*   **本地回环**: 服务默认只绑定 `127.0.0.1`，防止外部访问。
*   **认证**: Server 和扩展之间可以使用 Token 认证（配置于 `gateway.auth.token`）。
